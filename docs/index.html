웹 기반 스플랫 뷰잉과 제작 워크플로

SFAC Studio는 PLY를 브라우저에서 바로 불러와 Gaussian Splat을 탐색합니다. 별도 설치 없이 URL만으로 공유할 수 있어, 리뷰·발표·납품용 뷰어로 쓰기 좋습니다.

궤도(Orbit) 카메라와 그리드·축 기즈모로 장면을 읽고, 타임라인에서 카메라 키프레임과 재생으로 움직임을 설계합니다. 사각형·브러시·박스·구 등 볼륨 선택으로 객체를 고르고, 말풍선 형태의 설명 코멘트와 카메라 복원으로 협업 피드백을 남길 수 있습니다.

CAD(STEP/STP, IGES/IGS)도 동일한 로드 흐름으로 열 수 있어, 스플랫과 메시를 한 작업 공간에서 다루는 확장 워크플로를 지향합니다.

동작 방식

뷰어 코어, 타임라인, UI, 보내기 모듈이 나뉘어 있으며, PlayCanvas와 splat 파이프라인 위에서 동작합니다.

뷰어와 로딩

파일 선택·드래그 앤 드롭으로 PLY 및 CAD를 불러오고, 세션과 시퀀스 재생으로 여러 자산을 이어서 볼 수 있습니다. 무한 그리드와 성능 설정으로 대용량 장면도 조절 가능합니다.

타임라인과 카메라

키프레임, 재생, 카메라 이동·템플릿, 핀·틱·툴팁 등 타임라인 UI가 한데 묶여 있어, “보여줄 시점”을 정밀하게 맞출 수 있습니다.

보내기

MP4 영상, 단일 HTML 뷰어(코멘트 오버레이·기본 Fly 모드·쿼터니언 카메라 전환), PLY 및 압축 PLY까지 보내 배포와 아카이브에 활용할 수 있습니다.

선택 도구

사각형, 브러시, 박스·구 볼륨 셀렉터로 영역을 지정하고 히스토리로 되돌립니다.

설명 마커

말풍선 마커와 카메라 복원으로 코멘트를 장면에 고정해 협업합니다.

Fly 모드

WASD·Q/E·마우스로 이동하고, Orbit으로 복귀할 때 롤 뒤집힘 없이 slerp 전환합니다.

배포와 로컬 개발

공개 데모는 sfacstudio.vercel.app에서 확인할 수 있습니다. 소스는 sfacspace/sfacstudio-viewer 저장소에 있으며, Node.js 18 이상에서 npm install 후 npm run dev로 로컬 미리보기를 띄울 수 있습니다.

프로젝트 소개 페이지는 이 저장소의 docs/에 있으며, GitHub Pages에서 main 브랜치의 /docs 폴더를 소스로 지정하면 https://<org>.github.io/sfacstudio-viewer/ 형태로 호스팅할 수 있습니다.

UI 문자열은 영어·한국어 로케일을 지원해 팀 단위 사용에 맞출 수 있습니다.

데스크톱 전용 뷰어와의 차이

네이티브 뷰어는 GPU·파일 시스템에 직접 붙어 최고 성능을 내는 경우가 많습니다. SFAC Studio는 설치 없이 링크로 열리고, HTML·영상 보내기로 공유와 아카이브에 강점이 있습니다. 무거운 편집보다는 리뷰·시연·가벼운 타임라인 작업에 맞춘 트레이드오프입니다.

웹 뷰어의 강점

즉시 공유, 크로스 플랫폼, 단일 HTML 패키징, 브라우저만으로 협업 피드백(마커)까지.

고려할 점

브라우저·GPU·메모리 한계에 따라 초대형 씬은 성능 패널과 에셋 분할이 필요할 수 있습니다.
